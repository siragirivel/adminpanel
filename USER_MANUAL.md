# Sirigirvel User Manual

This manual is for workshop staff who use the Sirigirvel web app every day.
It explains how to use the system screen by screen in simple business language.

This is not a developer document.

## 1. What this app is used for

Sirigirvel helps a workshop team manage:

1. Vehicle registration and customer records
2. Service quotations
3. Final invoices and PDF downloads
4. Spare parts inventory
5. Spare part purchase orders
6. Day book income and expense entries
7. Cash and bank balance overview
8. Activity logs and low-stock alerts

## 4. Daily workflow in one view

For a normal workshop day, the usual order is:

1. Sign in
2. Register or search the vehicle
3. Add service notes and images
4. Create a quotation if approval is needed
5. Convert quotation to invoice, or create invoice directly
6. Download the invoice PDF
7. Track spare part stock
8. Add supplier orders when stock is purchased
9. Add manual day book entries for non-invoice income or expenses
10. Review balances in Accounts

## 11. Create a quotation

Use quotation when the customer must approve the estimated work first.

You can create a quotation from:

1. The Dashboard
2. The vehicle profile
3. The Quotation page

### Steps

1. Open `Quotation`
2. Click `New Quotation`
3. Select the vehicle
4. Add spare parts
5. Add labour rows
6. Set start date
7. Set end date
8. Enter discount if needed
9. Enter notes if needed
10. Save

### Sample quotation

Vehicle:

1. `SGV-2026-001`
2. `Rajan Kumar`
3. `TN37AB1234`

Parts:

1. `Front Brake Pad Set` qty `1`
2. `Engine Oil 5W30` qty `4`

Labour:

1. `Brake pad replacement` amount `800`
2. `General service labour` amount `1200`

Discount:

`500`

Validity:

1. Start date: `19-03-2026`
2. End date: `26-03-2026`

### Important quotation behavior

Quotation is only an estimate.

Quotation does not:

1. Reduce spare part stock
2. Update cash or bank balance

Quotation does:

1. Save the estimate
2. Generate a quotation PDF
3. Let you convert the estimate into an invoice later

## 12. View or download a quotation

Go to `Quotation` to see all saved quotations.

From the quotation list, you can:

1. Search by quotation number, customer, or registration number
2. Open a quotation
3. Download its PDF
4. Delete it
5. Convert it to an invoice

## 13. Convert quotation to invoice

When the customer approves the estimate:

1. Open `Quotation`
2. Find the quotation
3. Click the convert option
4. The invoice form opens with quotation data filled in
5. Check parts, labour, totals, and payment mode
6. Save the invoice

This is the safest way to avoid retyping the same work twice.

## 15. What happens automatically when an invoice is saved

When an invoice is saved, the app automatically:

1. Creates the invoice record
2. Downloads the invoice PDF
3. Reduces stock for selected spare parts
4. Creates an activity log entry
5. Creates a matching day book entry

Balance behavior:

1. `Cash` invoice increases Petty Cash
2. `UPI`, `Card`, and `Cheque` invoice increase Bank balance

## 19. Low stock notifications

The bell icon in the top bar shows notifications.

Low stock alerts appear there when a part's stock becomes less than or equal to its threshold.

Example:

1. Stock = `2`
2. Threshold = `3`

That part will show as a low-stock alert.

Use these alerts to reorder parts before they run out.

## 27. Recommended operating process for workshop staff

Use this standard process to keep records clean:

1. Search for the vehicle first before creating a new one
2. If the vehicle is new, register it fully
3. Add service note and photos during intake
4. Create a quotation if customer approval is pending
5. Convert the quotation to invoice after approval
6. If no quotation is needed, create invoice directly
7. Check that the payment mode is correct before saving the invoice
8. Use Day Book only for entries not already created by invoice saving
9. Record supplier purchases in Spare Orders
10. Add the expense to Day Book if that purchase should affect balances
11. Check Notifications for low stock
12. Use Logs to review important changes

## 28. Common mistakes to avoid

1. Do not create the same vehicle twice; search first
2. Do not use the wrong payment mode on an invoice; balances depend on it
3. Do not forget to set stock threshold for new spare parts
4. Do not assume Spare Orders change balances automatically
5. Do not create a manual Day Book entry for an invoice that already created one automatically

## 31. If something does not look right

Check these first:

1. Search whether the record already exists
2. Confirm that you are in the correct module
3. Refresh the page once
4. Confirm that the payment mode is correct
5. Confirm that the part exists in inventory before billing
6. Check Logs for recent changes

## 32. Final advice for daily use

The cleanest workflow is:

1. Vehicle first
2. Quotation if approval is needed
3. Invoice after approval
4. Inventory monitoring throughout the day
5. Day Book only for manual financial entries
6. Accounts for management review

If staff follow this order consistently, duplicate records and balance mistakes are much less likely.
